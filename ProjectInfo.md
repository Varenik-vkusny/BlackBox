# **Промпт для разработки: Фоновый Контекст-Демон "BlackBox"**

**Твоя роль:** Senior Rust Engineer / System Architect.
**Задача:** Разработать **BlackBox** — локальный MCP-сервер на Rust для пассивного сбора контекста разработки.

### **1. Концепция и "Why"**
Существующие AI-агенты (Claude Code, Cursor) — **реактивны**. Они видят только то, что происходит внутри их сессии. 
**BlackBox** — это "бортовой самописец" (Flight Recorder) рабочего окружения. Он **пассивно** агрегирует данные из терминалов, логов Docker и состояния Git в реальном времени. Когда разработчик обращается к ИИ, BlackBox предоставляет «моментальный снимок» того, что произошло *до* запроса и в *параллельных* процессах, избавляя от ручного копирования ошибок.

### **2. Системные ограничения (Hard Constraints)**
1.  **Strictly No IDE Focus:** Мы **не** отслеживаем открытые файлы или курсор. Фокус только на системных потоках (Terminal/Docker) и метаданных проекта.
2.  **Zero-Leak Security:** 
    *   Все значения в `.env` и конфигах маскируются (`<MASKED>`).
    *   Сырые логи оборачиваются в XML-теги безопасности для предотвращения Prompt Injection.
3.  **Self-Documentation:** Создай файл `improvements.md`. В него записывай все технические долги, потенциальные уязвимости и идеи по оптимизации, которые возникают в процессе реализации каждой фазы. Немного написал про это также в файле про первую фазу.

### **3. Технологический стек**
Ядро: Rust (Edition 2021, MSRV 1.77+).
Асинхронность: tokio (v1.37+).
MCP Транспорт: Стандартный stdio JSON-RPC (используем serde_json v1.0.115+ и serde v1.0.197+).
Хранение логов: Конкурентные lock-free каналы crossbeam (v0.8+) или современный асинхронный flume (v0.11+). Для кольцевого буфера используем ringbuffer (v0.3+).
Git: gix / Gitoxide (v0.62+) — чистый, безопасный и сверхбыстрый Rust-аналог (строгий отказ от C-биндингов git2/libgit2).
Парсинг: regex (v1.10+).
VS Code плагин (для MVP): TypeScript (v5.4+) + Node.js (v20+ LTS). VS Code Engine API (^1.87.0).

### **4. План реализации**

#### **🚀 Фаза 1: Базовый MCP и Интеграция с VS Code (MVP)**
**Цель:** Работающий MCP-сервер с накоплением логов из VS Code.

*   **Шаг 1.1: MCP Core.** Реализация протокола (initialize, tools/list, tools/call).
*   **Шаг 1.2: Snapshot Tool.** Реализация `get_snapshot()`: статус демона, текущая ветка (через `gix`), краткая статистика буфера.
*   **Шаг 1.3: Project Metadata Tool.** `get_project_metadata()`: анализ зависимостей (Cargo, npm, go) и маскированный экспорт имен переменных из `.env`.
*   **Шаг 1.4: Terminal Bridge.** 
    *   В Rust: TCP-листенер для приема строк и сохранения последних 5000 строк в буфер.
    *   В TS: VS Code плагин, пересылающий данные из `window.onDidWriteTerminalData`.
*   **Шаг 1.5: Context Tool.** `get_terminal_buffer()`: отдает последние 100 строк, очищенных от ANSI-кодов через `regex`, обернутых в XML-теги безопасности.
    *   *Note:* Добавь Regex-парсер стектрейсов с меткой `// @TODO [Future Upgrade]` для замены на tree-sitter.

#### **⚡ Фаза 2: Инфраструктура и Smart-аналитика**
*   **Docker SDK:** Потоковое чтение `/var/run/docker.sock` (`bollard`), фильтрация только `ERROR/WARN`.
*   **Smart Diffs:** Генерация сводок изменений Git, пересекающихся с ошибками в логах.
*   **Log Mining:** Реализация алгоритмов кластеризации (Drain/Spell) для сжатия цикличного спама ошибок.

#### **🛡 Фаза 3: Native OS & Privacy**
*   **PTY Interception:** Переход на системный перехват (Linux PTY / Windows ConPTY).
*   **Zero-Copy:** Оптимизация буфера через паттерн LMAX Disruptor.
*   **PII Masking:** Интеграция локальной ML-модели (`candle`) для маскирования личных данных на CPU.
