# 07. Testing and Security

> [!IMPORTANT]
> **Repomix Context Command:**
> `repomix --include "crates/blackbox-test/**,crates/blackbox-daemon/src/mcp/tools.rs" --output security_context.txt`

Безопасность и надежность — критические требования для системы, которая агрегирует логи разработки.

## Integration Testing (`blackbox-test`)
Вместо обычных Unit-тестов, BlackBox полагается на полноценные интеграционные сценарии:
1.  **Subprocess Execution**: Тестовый фреймворк компилирует и запускает `blackbox-daemon.exe` в отдельном процессе.
2.  **Protocol Verification**: Тесты симулируют работу ИИ, отправляя JSON-RPC запросы в `stdin` демона и проверяя ответы в `stdout`.
3.  **Bridge Injection**: Тесты открывают TCP соединение с портом 8765 и "впрыскивают" тестовые данные (например, панику Rust), после чего проверяют, смог ли MCP-инструмент `get_compressed_errors` правильно распознать структуру ошибки.
4.  **11 Секций тестирования**: Покрывают всё — от инициализации протокола до корректной обработки разрывов TCP-соединения.

## Security Layers

### 1. Path Traversal Protection
Инструмент `read_file` использует строгую проверку путей:
*   Путь нормализуется через `canonicalize()`.
*   Проверяется, что результирующий путь начинается с рабочей директории проекта (`cwd`).
*   Попытки выйти за пределы проекта (например, через `../../etc/passwd`) блокируются с кодом ошибки `-32602`.

### 2. PII Masking & Privacy (`pii_masker.rs`)
Входящие логи проходят через многослойный фильтр перед попаданием в буфер:
*   **Regex Scan**: Маскирует явные паттерны: email, JWT, Bearer-токены, номера кредитных карт.
*   **Secret Detection**: Ищет конструкции типа `password=SECRET`, `API_KEY: value`.
*   **Entropy Scanner**: Вычисляет энтропию Шеннона для каждого токена длиной более 20 символов. Если энтропия выше **4.5**, токен считается случайным ключом/хешем и заменяется на `<SECRET_MASKED>`.

### 3. XML Guard (`typed_context.rs`)
Данные в MCP-ответах оборачиваются в XML-теги (например, `<terminal_output>`).
*   Чтобы предотвратить **Prompt Injection** (когда данные из терминала пытаются управлять поведением ИИ), все закрывающие теги и потенциально опасные конструкции (типа `<script>`) внутри контента экранируются.
*   Каждый блок помечается атрибутами `untrusted="true"` и `pii_masked="true"`, давая ИИ понять, как именно стоит интерпретировать эти данные.
