# 05. Lab and UI

> [!IMPORTANT]
> **Repomix Context Command:**
> `repomix --include "blackbox-lab/src/**,crates/blackbox-tui/src/**,crates/blackbox-sandbox/src/**" --output ui_context.txt`

Для мониторинга работы BlackBox и проверки того, что видит ИИ, реализованы три интерфейса.

## BlackBox Lab (Web Interface)
Веб-интерфейс, написанный на **Vite + React**. Он предназначен для визуального анализа накопленного контекста.
*   **Grid Dashboard**: Отображает состояние демона, последние логи терминала, кластеры ошибок и статус Docker.
*   **Live Updates**: Опрашивает административный API демона для обновления данных в реальном времени.
*   **Visualization**: Позволяет удобно просматривать сложные структуры, такие как стек-трейсы и диффы, в отрыве от текстового интерфейса ИИ.

## TUI Dashboard (`blackbox-tui`)
Консольный интерфейс на базе **Ratatui**.
*   **Purpose**: Быстрый мониторинг процесса без необходимости открывать браузер.
*   **Metrics**: Показывает загрузку кольцевого буфера, uptime, текущую ветку Git и наличие недавних ошибок.
*   **Connectivity**: Общается с демоном через Status Server (порт 8766).

## Sandbox (`blackbox-sandbox`)
Специальный инструмент для отладки MCP-инструментов.
*   **Tool Testing**: Позволяет вручную вызывать любой из 9 инструментов MCP и видеть сырой JSON-ответ.
*   **Postmortem Exploration**: Имеет вкладки для визуализации таймлайна инцидентов и корреляции логов.
*   **Development tool**: Используется разработчиками BlackBox для верификации логики фолбэков и фильтрации.

## Communication Architecture
Визуальные интерфейсы используют два способа общения с демоном:

1.  **Status Server (TCP :8766)**: Легковесный сервер, который при подключении сразу отдает текущий статус системы (JSON) и закрывает соединение. Используется TUI.
2.  **Admin API (HTTP :8768)**: Полноценный Axum-сервер с набором эндпоинтов (`/api/status`, `/api/terminal`, `/api/inject`). Поддерживает CORS для работы из веб-браузера. Используется BlackBox Lab.
3.  **Terminal Bridge (TCP :8765)**: Принимает входящие логи. Лаборатория может использовать этот порт для "инъекции" тестовых логов (через `/api/inject` проксируемый в демон).
